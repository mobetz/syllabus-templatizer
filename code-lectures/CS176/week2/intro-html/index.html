HTML: Tags and Attributes

HTML stands for HyperText Markup Language.

A markup language is a system for annotating content with additional information that may convey formatting, presentation or the structure of data. Hypertext refers to the links that connect pages on the web.

One of the common prototypical markup languages that's used by many applications is XML. HTML is actually a derivative of XML (the eXtensible Markup Language.)

In HTML, markup is indicated using "tags". A tag is just a word of text surrounded by <angle brackets>. The most basic tag is the <html > tag, which identifies a document as being html.

Most tags in HTML identify regions on a page: they come in pairs, and indicate where a certain feature of the document begins, and also where it ends. A closing tag will have an identical name to its opening tag, but it will be prefixed by a /. These tags will always end in last-opened first-closed order, almost like nesting dolls.

Inside the html tag, there are two main sections of the html document: a HEAD, and a BODY.<

The BODY represents everything about the page we interact with inside the tab.

The HEAD holds extra metadata about the page that does not get displayed directly, but may be useful for the browser, assistive technologies, or a search engine.

Notice how our rendered website isn't respecting our whitespace? We have to identify where paragraphs start and end using <p> tags. Similarly, we can use <h1> through <h6> to represent different levels of headers and subheaders.

We can emphasize important terms with <em> or <strong>.

Images are specified using the <img> tag. However, images also work just a little bit differently. Some tags like images require more information than we could give them using just our "nesting doll" positioning. For these types of tags, we can use "attributes": Attributes are key-value pairs that identify additional features about the content inside the tag. To add an attribute, we use the name, an equals sign, then the value "inside quotation marks".

The img tag requires a src (source) attribute, which points to the link where the image is located, and accepts other attributes for sizing information like height and width:

One last major structural tag is the hyperlink. Hyperlinks are created using the <a> tag. A is short for "anchor", because these anchors tie two different places on the web together. Typically, an anchor tag is used with the href attribute, which has a full path to another website. If you want to link to another file the same folder, then you can just use the file's name.

There is another way to use the anchor tag: When used with the name or id attribute, an anchor can also save a position on a page. Then, if another anchor's href uses that name with a #, we can instantly navigate back to the anchor.

One final note on writing HTML: if you want to type something out that HTML would incorrectly render as a tag, we can do so using escape codes.Escape codes are specified with an ampersand (&) and ending with a semicolon (;). For instance, we can use & to refer to ampersand, and and &lt; (less than) to refer to our opening bracket in our tag.

Semantic Tags

Most tags in HTML don't actually do anything at all, beyond setting some basic positional data, and maybe having a bit of default styling.

We even have two proto-tags that represent this behavior with no extra meaning attached:

<div> represents a block in the document with no additional meaning.

<span> represents an inline segment of text with no additional meaning.

If we wanted to, we could go through and replace every one of our paragraph, header, strong, and emphasis tags with a bunch of divs and spans. So why do we bother learning about so many more?

If we only had divs and spans, our webpage would end up looking something like this:

<div></div> <div> <span>Welcome to my page</span> <div> <div> <img src="..."> <div>Look at this cool image</div> </div> </div> </div> <div>Copyright 2020</div>

Looking at a page like this, it's nearly impossible to figure out what any of those divs represent. What's worse, if we were trying to parse the HTML not as a human, but instead as a machine, we'd have no way of knowing what any of these divs meant and which were important.

A lot of elements, such as <section>, <footer>, and even <p> serve mainly as clues for ourselves, other developers, and machines to help give meaning to the structure of our document. Imagine instead our document above looked like this:

<header></header> <section> <h3>Welcome to my page</h3> <article> <figure> <img src="..."> <figcaption>Look at this cool image</figcaption> </figure> </article> </section> <footer>Copyright 2020</footer>

Now, it becomes clear which parts of the page are organizational, and which have the main content we actually care about.The practice of using meaningful tags that describe not just the layout, but also the purpose of different parts of the page is known as Semantic HTML, and the introduction of semantics into our HTML documents has been one of the biggest pushes in modern HTML standards.
